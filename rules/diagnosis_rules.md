# JFFイベント診断ルール定義（人間可読版）

作成日: 2026-05-18  
対象: `06_PROJECTS/EVENT_DIAGNOSIS/index.html` の診断ロジック

このドキュメントは、`index.html` 内 JavaScript の `RULES_EVENT` / `RULES_VENUE` / `RULES_CAPACITY` / `RULES_FLAGS` 定数と1対1で対応している。  
**ルールを更新したい時は、まずこのMDで議論し、合意後に `index.html` のJS定数に反映する**。

## 診断エンジンの構造

入力（フォーム）→ 5つのレイヤーでルールを合成 → 出力（4カテゴリ）

```
入力:
  - eventType       (talk / screening / music)
  - capacity        (s30 / s80 / s200 / sLarge)
  - venueType       (cinema / cafe / space / outdoor)
  - env             (indoor / outdoor / semi)
  - merch / food / qa / stream / lateNight (フラグ)
  - eventDate       (任意)

合成レイヤー:
  1. イベントタイプ別ベース
  2. 会場タイプ別追加
  3. キャパ別追加（+スタッフ配置を上書き）
  4. 環境（屋外/半屋外なら屋外ルール強制適用）
  5. オプションフラグ別追加
  6. 開催日が30日未満なら時間リスクを追加

出力:
  - equipment: [{ name, note }]   必要機材
  - risks:     [{ name, note }]   主要リスク
  - venue:     [{ name, note }]   会場確認リスト
  - staff:     { heads, roles, note }  スタッフ配置
```

重複は `name` でユニーク化される（同名項目は最初の1つだけ採用）。

## 1. イベントタイプ別ベースルール

### トークイベント
| カテゴリ | 項目 |
|---------|------|
| 必要機材 | ワイヤレスマイク（登壇者人数+1本）/ スピーカー（最低2本）/ 登壇者用テーブル・椅子・水 / タイムキープ時計 / ネームプレート |
| 主要リスク | 登壇者ドタキャン / マイクトラブル / 進行押し |
| 会場確認 | 壇上の照明 / 音響卓・ミキサー / 登壇者控室 |

### 映画上映
| カテゴリ | 項目 |
|---------|------|
| 必要機材 | プロジェクター(3500lm+) / スクリーン / 再生機材(DCP/Blu-ray/PC) / ステレオ以上音響 / 遮光対策 / HDMI・変換アダプタ |
| 主要リスク | 素材不具合 / 遮光不足 / 音響調整漏れ / 上映権手続き |
| 会場確認 | スクリーン有無・サイズ / 遮光性能 / 音響卓出力規格 / 投影距離 |

### DJ・音楽
| カテゴリ | 項目 |
|---------|------|
| 必要機材 | PA(メイン+サブウーファー) / DJブース / モニタースピーカー / ミキサー(4ch+) / ケーブル類 / 照明2系統 |
| 主要リスク | 近隣騒音クレーム / 著作権処理(JASRAC) / 電源容量オーバー / 演者機材トラブル / 入退場混雑 |
| 会場確認 | 電源容量 / 防音性能 / 搬入経路 / 撤収時間 |

## 2. 会場タイプ別追加ルール

### 映画館・ホール (cinema)
- リスク: DCP納品期限（2週間前）/ オペレーター対応時間制約
- 会場確認: 会場機材ヒアリングシート / 受付ロビー・物販可否 / 館内アナウンス

### カフェ・バー (cafe)
- 機材: 持ち込みプロジェクター・スクリーン
- リスク: 電源容量不足 / 通常営業との両立 / 近隣騒音
- 会場確認: ブレーカー容量 / 暗幕・遮光 / 厨房ノイズ・匂い / 貸切時間と撤収

### イベントスペース (space)
- 機材: 持ち込み機材一式
- リスク: 搬入経路の制約 / 原状回復範囲
- 会場確認: 施設利用申請書 / 搬入経路 / 原状回復ルール / ゴミ処理

### 屋外・特殊 (outdoor)
※ 会場タイプ「屋外・特殊」を選んでも、環境「屋外」を選んでも同じルールが適用される。
- 機材: 発電機 or 大容量ポータブル電源 / テント・タープ / 照明
- リスク: 天候プランB / イベント保険 / 通信環境 / 近隣告知
- 会場確認: 電源確保手段 / 代替会場 / トイレ・水道 / 駐車場・導線 / ゴミ処理

## 3. キャパ別ルール（スタッフ配置を決定）

| キャパ | 最低スタッフ数 | 配置 | 補足 |
|--------|---------------|------|------|
| 〜30名 (s30) | 3名 | 受付兼物販 ×1 / 運営 ×1 / 技術 ×1 | 兼務前提、最小チーム |
| 31〜80名 (s80) | 5名 | 受付 ×2 / 運営 ×1 / 技術 ×1 / 物販 ×1 | 受付ピーク注意 |
| 81〜200名 (s200) | 8名 | 受付 ×3 / 運営 ×2 / 技術 ×1 / 物販 ×1 / 誘導 ×1 | 客席誘導と物販ピーク管理が鍵 |
| 201名〜 (sLarge) | 12名+ | 受付 ×4 / 運営 ×3 / 技術 ×2 / 物販 ×2 / 誘導 ×2 / 警備 ×1 | **個別設計を強く推奨** |

`s200` 以上で追加機材ルール（受付用タブレット・複数台名簿）が発火。  
`sLarge` で「規模特有のリスク多数」警告が表示される。

## 4. オプションフラグ別追加ルール

### 物販あり (merch)
- 機材: 物販テーブル・釣銭・キャッシュトレイ / 決済端末
- 会場確認: 物販エリアの動線

### 飲食提供あり (food)
- 機材: 保冷・保温機材 / 提供用テーブル
- リスク: 食品衛生・営業許可 / アレルギー表示
- 会場確認: 厨房・給水排水 / ゴミ処理（生ゴミ含む）

### Q&A・トークパート (qa)
- 機材: 客席用ワイヤレスマイク ×2本以上
- リスク: 質問が出ない / 偏る

### 配信・収録あり (stream)
- 機材: カメラ×2 / スイッチャー・配信PC / マイク入力ライン / 安定回線
- リスク: 回線断絶 / 肖像権・配信許諾
- 会場確認: 有線LAN or 専用Wi-Fi

### 深夜営業（22時以降） (lateNight)
- リスク: 風営法該当の可能性 / 終電・タクシー導線
- 会場確認: 深夜営業許可

## 5. 環境ルール

| 環境 | 効果 |
|------|------|
| 屋内 (indoor) | 追加ルールなし |
| 屋外 (outdoor) | 「屋外・特殊」会場ルールを強制適用 |
| 半屋外 (semi) | 「屋外・特殊」会場ルールを強制適用 |

---

## 6. 開催日ルール

| 条件 | 効果 |
|------|------|
| 開催まで30日未満 | リスクに「準備期間が短い」警告を追加 |
| 開催日が過去 | リスクに「入力ミス確認」警告を追加 |
| 30日以上 or 未入力 | 追加なし |

## ルール更新ワークフロー

1. このMDで案を編集（追加/修正/削除）し、運営チームでレビュー
2. 合意後、`index.html` の対応するJS定数（`RULES_EVENT` 等）を更新
3. ブラウザで3パターン（トーク+小規模、上映+カフェ、音楽+屋外）を再診断し、想定どおり出るか確認
4. README.mdの更新ログに変更内容を追記

---

## 既知の限界（運用しながら拡充）

- 音楽/DJ・トーク系の知見は既存ドキュメントに薄く、**MVPは暫定ルール**。実イベント後の振り返りで増やす。
- 機材の数量（W数・本数）は会場サイズに連動する細かい計算をしていない（概算のみ）。
- 予算・収支は診断対象外（既存の収支ダッシュボードで対応）。
- 多言語イベント・国際イベント特有のルール（通訳・字幕等）は未対応。
- 「会場決定後の本気のチェック」には `04_SCREENINGS/テンプレート_会場タイプ別チェックリスト.md` を併用すること。
