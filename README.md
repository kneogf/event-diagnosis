# JFF イベント診断ツール

最終更新: 2026-05-18  
ステータス: **MVP v0.1（最小構成）**

---

## 概要

JFFの主催・協力するイベント（トーク / 映画上映 / DJ・音楽）について、主催者が **「自分のイベントに何が必要か」をブラウザで開いた1画面で把握できる** ようにする診断ツール。

入力:
- イベントタイプ（3種）
- 想定キャパ（4段階）
- 会場タイプ（4種）
- 屋内/屋外
- オプション要素（物販・飲食・Q&A・配信・深夜営業）
- 開催予定日（任意）

出力:
- 必要機材リスト
- 主要リスク
- 会場確認リスト
- スタッフ配置（最低人数・役割）
- 有料レビュー導線（CTA）
- PDF印刷

---

## 使い方

1. `index.html` をブラウザで開く（ダブルクリック or `open index.html`）
2. フォームに入力 → 「診断する」
3. 診断結果が画面下部に展開
4. 「PDFで保存・印刷」でブラウザの印刷ダイアログ → PDF出力可能
5. 「専門家による個別レビューを依頼する」ボタンで `mailto:` が起動（CTA）

ローカルファイル運用なので、サーバー不要・ビルド不要・依存なし。

## 問い合わせフォーム → スプレッドシート連携

LP内の問い合わせフォームは、Google Apps Script経由でGoogleスプレッドシートに行追加されます。

**初回セットアップ必須**: [docs/APPS_SCRIPT_SETUP.md](docs/APPS_SCRIPT_SETUP.md) の手順に従って、スプレッドシート作成 → Apps Scriptデプロイ → `index.html`の`APPS_SCRIPT_URL`定数を更新してください（所要10分）。

未設定の場合、フォーム送信時に「送信先が未設定です」と表示されます。

---

## ロジックの場所

- **ルールデータ**: `index.html` 冒頭の JS 定数
  - `RULES_EVENT`（イベントタイプ別）
  - `RULES_VENUE`（会場タイプ別）
  - `RULES_CAPACITY`（キャパ別・スタッフ配置）
  - `RULES_FLAGS`（オプション要素別）
- **診断エンジン**: `function diagnose(input)` 内で全レイヤーを合成
- **人間可読版**: `rules/diagnosis_rules.md` を参照

---

## 更新方法

1. `rules/diagnosis_rules.md` でルール案を編集（運営チームでレビュー）
2. `index.html` の対応するJS定数を更新
3. ブラウザで3パターン以上で動作確認
4. このREADMEの「更新ログ」に追記

---

## 既知の限界 / 今後の拡張

### MVPで意図的にやらないこと
- デザインの作り込み（実用性優先）
- 機材の数量計算（概算表記のみ）
- 予算・収支シミュレーション（既存の収支ダッシュボードで対応）
- サーバー連携・データ保存

### 将来拡張（やるなら）
- 診断結果から `上映会OS_起動` プロンプトを自動生成
- Markdown形式で結果ダウンロード（プロジェクトフォルダに保存）
- Next.jsコーポレートサイトへの統合（SEO・URL共有可能化）
- 結果データを蓄積して傾向分析
- 有料レビューの決済導線（Stripe等）
- 音楽/DJ・トーク特有ルールの拡充（実イベント振り返りから）

## 有料CTAの設定変更

`index.html` 内の `<a id="ctaLink" ...>` の `href` を差し替える：
- mailto: の場合: 件名・本文をURLエンコードして指定
- Google Form の場合: フォームURLに置換
- 決済リンクの場合: Stripe Payment Link 等のURLに置換
